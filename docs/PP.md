# Tallia — PP (Privacy Policy / 개인정보처리방침)

> 개인정보 보호법 제30조 및 개인정보 처리방침 작성지침(2025.4. 개정) 기반 작성
> Tallia 서비스에서 처리하는 개인정보의 항목, 목적, 보유기간, 파기, 안전성 확보조치, 정보주체 권리 등을 규정한다.

---

## 제1조 (개인정보의 처리 목적)

Tallia는 다음의 목적을 위하여 개인정보를 처리합니다. 처리하는 개인정보는 다음의 목적 이외의 용도로 이용되지 않으며, 이용 목적이 변경되는 경우에는 개인정보 보호법 제18조에 따라 별도의 동의를 받는 등 필요한 조치를 이행합니다.

| 목적 | 상세 |
|------|------|
| 회원 가입 및 관리 | 회원 가입 의사 확인, 회원 식별·인증, 회원 자격 유지·관리, 서비스 부정이용 방지 |
| 서비스 제공 | 대학 입학전형 평가점수 산출(원점수·환산점수), 엑셀 데이터 업로드·검증·계산·결과 조회·다운로드, 평가 설정 관리 |
| 수험생 데이터 처리 | 평가위원 점수 집계, 자동 채점, 문항 점수 계산, 매핑 테이블 변환 등 대학별 평가 유형에 따른 점수 산출 |
| 감사 및 기록 관리 | 데이터 접근·수정·다운로드 이력 기록, 채점 검증 로그, 업로드 이력 관리 |

## 제2조 (처리하는 개인정보의 항목)

### 2.1 회원 정보 (직접 수집)

| 구분 | 항목 | 수집 방법 |
|------|------|-----------|
| 필수 | 이메일, 비밀번호, 성명 | 회원가입 시 이용자 입력 |
| 자동 수집 | IP 주소, 접속 일시, 서비스 이용 기록 | 서비스 이용 과정에서 자동 생성·수집 |

### 2.2 수험생 데이터 (대학 관리자가 업로드)

| 구분 | 항목 | 수집 방법 |
|------|------|-----------|
| 공통 필수 | 수험번호, 성명 | 대학 관리자가 엑셀 파일로 업로드 |
| A유형 | 평가위원별 항목별 점수 또는 등급 | 엑셀 업로드 |
| B유형 | 시험유형, 문항별 답안 | 엑셀 업로드 |
| C유형 | 문항별 점수 (채점위원별) | 엑셀 업로드 |
| D유형 | 자격유형·급수, 어학유형·점수, 평점 등 | 엑셀 업로드 |

> **수험생 데이터의 특수성**: 수험생 데이터는 Tallia가 직접 수집하지 않으며, 대학(개인정보처리자)이 자체 수집한 데이터를 Tallia 시스템에 업로드하여 처리합니다. Tallia는 대학으로부터 개인정보 처리를 위탁받은 수탁자의 지위에 해당하며, 위탁 범위 내에서만 처리합니다.

### 2.3 처리하지 않는 항목

- 주민등록번호, 여권번호 등 고유식별정보
- 건강정보, 범죄경력 등 민감정보
- 14세 미만 아동의 개인정보 (서비스 대상이 아님)

## 제3조 (개인정보의 처리 및 보유 기간)

Tallia는 법령에 따른 보유기간 또는 정보주체로부터 동의받은 기간 내에서 개인정보를 처리·보유합니다.

| 대상 | 보유 기간 | 근거 |
|------|-----------|------|
| 회원 정보 | 회원 탈퇴 시까지 | 정보주체 동의 |
| 수험생 데이터 (점수, 업로드 원본) | 대학별 설정 기간 (기본 3년) | 고등교육법 시행령 제35조 (입학전형자료 보관), 대학별 학칙 |
| 계산 결과 (원점수, 환산점수, 중간 결과) | 수험생 데이터와 동일 | 상동 |
| 감사 로그 (접근·수정·다운로드 이력) | 3년 | 개인정보 보호법 제34조의2 (접속기록 보관) |
| 접속 기록 | 1년 이상 | 개인정보의 안전성 확보조치 기준 제8조 |

> **대학별 보관 기간 설정**: 각 대학(테넌트)은 자체 학칙 및 규정에 따라 데이터 보관 기간을 1~10년 범위에서 설정할 수 있습니다. 기본값은 3년이며, 보관 기간 경과 시 지체 없이 파기합니다.

## 제4조 (개인정보의 파기 절차 및 방법)

### 4.1 파기 절차

1. 보유 기간 경과 또는 처리 목적 달성 시 파기 대상 선정
2. 개인정보 보호책임자의 승인
3. 지체 없이 파기 실행 (보유기간 경과 시 정당한 사유가 없는 한 5일 이내)

### 4.2 파기 방법

| 대상 | 방법 |
|------|------|
| 데이터베이스 내 개인정보 | **완전 삭제** (DELETE + 관련 테이블 CASCADE). 소프트 삭제 미사용 |
| 업로드된 엑셀 원본 데이터 | 데이터베이스에서 완전 삭제 (JSONB 데이터 포함) |
| 계산 결과 및 중간 결과 | 수험생 데이터와 함께 완전 삭제 |
| 백업 데이터 | 백업 주기에 따라 자동 덮어쓰기 또는 수동 삭제 |

> **완전 삭제 원칙**: 삭제된 데이터는 복구가 불가능하며, 이는 PRD에 명시된 "데이터 삭제 시 복구 불가능하도록 완전 삭제" 요구사항을 준수합니다.

### 4.3 대학 관리자의 수동 삭제

대학 관리자는 보관 기간과 무관하게 언제든 평가 데이터(수험생 데이터 포함)를 삭제할 수 있으며, 삭제 시 관련된 모든 데이터(업로드 원본, 계산 결과, 중간 결과)가 함께 완전 삭제됩니다.

## 제5조 (개인정보의 제3자 제공)

Tallia는 원칙적으로 정보주체의 개인정보를 제1조에서 명시한 목적 범위 내에서 처리하며, 다음의 경우를 제외하고는 제3자에게 제공하지 않습니다.

1. 정보주체의 별도 동의가 있는 경우
2. 법률에 특별한 규정이 있거나 법령상 의무를 준수하기 위하여 불가피한 경우
3. 공공기관이 법령 등에서 정하는 소관 업무의 수행을 위하여 불가피한 경우

> 수험생 데이터는 해당 대학(테넌트) 내에서만 접근 가능하며, 다른 대학이나 외부 기관에 제공되지 않습니다.

## 제6조 (개인정보 처리의 위탁)

| 수탁자 | 위탁 업무 | 보유 기간 |
|--------|-----------|-----------|
| (클라우드 인프라 제공자) | 서버 운영 및 데이터 저장 | 위탁 계약 종료 시까지 |
| (이메일 발송 서비스) | 이메일 인증, 비밀번호 재설정 발송 | 발송 완료 후 즉시 삭제 |

> 위탁 업체는 서비스 운영 과정에서 확정 후 본 방침에 업데이트됩니다. 위탁 계약 시 개인정보 보호 관련 법규 준수, 개인정보 비밀 유지, 재위탁 제한, 사고 시 손해배상 책임 등을 명시합니다.

## 제7조 (정보주체와 법정대리인의 권리·의무 및 행사방법)

### 7.1 정보주체의 권리

정보주체(회원)는 다음의 권리를 행사할 수 있습니다.

| 권리 | 내용 |
|------|------|
| 열람 요구 | 본인의 개인정보 처리 현황 열람 |
| 정정·삭제 요구 | 부정확한 개인정보의 정정 또는 삭제 |
| 처리정지 요구 | 개인정보 처리의 정지 |
| 동의 철회 | 개인정보 수집·이용 동의 철회 (회원 탈퇴) |

### 7.2 행사 방법

- 서면, 전자우편 또는 서비스 내 설정 페이지를 통해 행사
- 요구일로부터 10일 이내 처리
- 대리인을 통한 행사 시: 개인정보 보호법 시행규칙 별지 제11호 서식에 따른 위임장 제출

### 7.3 수험생의 권리

수험생 데이터에 대한 열람·정정·삭제·처리정지 요구는 해당 수험생이 소속 대학의 입학처에 직접 요청해야 합니다. Tallia는 수탁자로서 대학(위탁자)의 지시에 따라 처리합니다.

## 제8조 (개인정보의 안전성 확보조치)

개인정보 보호법 제29조 및 개인정보의 안전성 확보조치 기준(고시)에 따라 다음과 같은 조치를 취합니다.

### 8.1 관리적 조치

| 조치 | 내용 |
|------|------|
| 내부 관리 계획 수립·시행 | 개인정보 보호 내부 관리 계획 수립 및 연 1회 이상 점검 |
| 접근 권한 관리 | 개인정보처리시스템에 대한 접근 권한을 업무 수행에 필요한 최소한의 범위로 제한 |
| 접근 권한 기록 | 접근 권한 부여·변경·말소 내역 3년 이상 보관 |
| 취급 직원 교육 | 개인정보를 취급하는 직원에 대한 정기 교육 실시 |

### 8.2 기술적 조치

| 조치 | 내용 |
|------|------|
| 비밀번호 암호화 | bcrypt 알고리즘 (salt rounds 12)으로 단방향 암호화 저장 |
| 전송 구간 암호화 | HTTPS(TLS 1.2 이상) 필수 적용 |
| 접속 기록 보관 | 모든 데이터 접근·수정·다운로드에 대한 감사 로그 기록, 1년 이상 보관 |
| 인증 보안 | JWT 기반 인증, Access Token 1시간 만료, Refresh Token Rotation |
| 멀티테넌트 격리 | tenant_id 기반 데이터 격리, TenantGuard로 다른 대학 데이터 접근 차단 |
| 커스텀 수식 보안 | 사용자 수식은 화이트리스트 기반 샌드박스에서 실행 (코드 인젝션 방지) |
| SQL Injection 방지 | ORM(Prisma) 파라미터 바인딩, Raw Query 사용 금지 |
| XSS 방지 | React 기본 이스케이프 + Helmet 미들웨어 |
| 보안 패치 | 의존성 패키지 정기 보안 업데이트 |

### 8.3 물리적 조치

| 조치 | 내용 |
|------|------|
| 서버 보안 | 클라우드 인프라 제공자의 물리적 보안 정책 준용 |
| 접근 통제 | 서버 접근은 SSH 키 기반 인증, IP 화이트리스트 적용 |

## 제9조 (개인정보의 자동 수집 장치의 설치·운영 및 거부)

### 9.1 쿠키 사용

Tallia는 인증 토큰 관리를 위해 쿠키를 사용할 수 있습니다.

| 항목 | 내용 |
|------|------|
| 사용 목적 | 인증 상태 유지 (JWT Refresh Token 저장) |
| 설치 항목 | 인증 관련 쿠키 (HttpOnly, Secure, SameSite) |
| 거부 방법 | 브라우저 설정에서 쿠키 저장 거부 가능 (이 경우 로그인 유지 불가) |

### 9.2 행태 정보

Tallia는 맞춤형 광고 등을 위한 행태 정보를 수집하지 않습니다.

## 제10조 (자동화된 의사결정에 관한 사항)

Tallia는 다음과 같은 자동화된 처리를 수행합니다.

| 처리 | 내용 | 정보주체 영향 |
|------|------|---------------|
| B유형 자동 채점 | 정답지 대조를 통한 문항별 자동 채점 | 수험생 점수에 직접 영향 |
| 파이프라인 계산 | 설정된 블록 순서에 따른 점수 산출 | 수험생 점수에 직접 영향 |
| 과락 판정 | 기준 미달 시 자동 과락 플래그 설정 | 수험생 합격 여부에 간접 영향 |

> **투명성 보장**: 모든 자동화된 처리의 중간 결과를 단계별로 확인할 수 있으며, B유형 채점 결과는 감사 로그에 기록됩니다. 자동 채점 결과에 대해 대학 관리자가 수동 검증할 수 있으며, 출제 오류 시 전원정답 처리 또는 배점 제외 등의 보정 조치가 가능합니다.

## 제11조 (개인정보 보호책임자)

| 구분 | 정보 |
|------|------|
| 직책 | 개인정보 보호책임자 |
| 성명 | (서비스 운영 시 지정) |
| 연락처 | (서비스 운영 시 지정) |
| 이메일 | (서비스 운영 시 지정) |

> 개인정보 처리에 관한 불만 처리 및 피해 구제 등을 위하여 개인정보 보호책임자를 지정하고 있으며, 정보주체는 서비스 이용 중 발생하는 모든 개인정보 보호 관련 문의, 불만 처리, 피해 구제 등에 관한 사항을 개인정보 보호책임자에게 문의할 수 있습니다.

## 제12조 (권익침해 구제방법)

정보주체는 개인정보침해로 인한 구제를 받기 위하여 다음의 기관에 분쟁 해결이나 상담 등을 신청할 수 있습니다.

| 기관 | 연락처 | 홈페이지 |
|------|--------|----------|
| 개인정보분쟁조정위원회 | 1833-6972 | privacy.go.kr |
| 개인정보침해신고센터 | 118 | privacy.kisa.or.kr |
| 대검찰청 사이버수사과 | 1301 | spo.go.kr |
| 경찰청 사이버수사국 | 182 | ecrm.police.go.kr |

## 제13조 (개인정보 처리방침의 변경)

이 개인정보처리방침은 시행일로부터 적용되며, 법령 및 방침에 따른 변경 내용의 추가, 삭제 및 정정이 있는 경우에는 변경사항의 시행 7일 전부터 서비스 내 공지사항을 통하여 고지합니다.

| 항목 | 내용 |
|------|------|
| 시행일 | (서비스 출시 시 확정) |
| 버전 | 1.0 |

```
    ├── 회원가입 → 이메일, 비밀번호, 성명 수집
    │
    └── 엑셀 업로드 → 수험생 데이터 (수험번호, 성명, 점수/답안/등급 등)
            │
            ├── 데이터 검증 → 검증 결과 표시
            ├── 계산 실행 → 원점수 + 환산점수 산출 (중간 결과 저장)
            ├── 결과 조회 → 감사 로그 기록
            └── 엑셀 다운로드 → 다운로드 이력 기록
                    │
                    └── 보관 기간 경과 → 완전 삭제 (CASCADE)
```
