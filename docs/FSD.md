## 평가 유형 (파이프라인)

### 커스텀 블록
사용자가 커스텀 연산 블록을 추가할 수 있다.
비전공자를 위해 **템플릿 방식**을 기본으로, 고급 사용자용 수식 입력도 지원.

**커스텀 블록 템플릿:**

| 템플릿 | 설명 | 파라미터 예시 |
|--------|------|---------------|
| 가산점 부여 | 조건 충족 시 점수 가산 | 조건: 점수 >= 80, 가산: +5점 |
| 비율 조정 | 점수에 비율 곱하기 | 비율: 0.6 |
| 구간 변환 | 점수 구간별 다른 값 매핑 | 90~100→A, 80~89→B, ... |
| 상한/하한 제한 | 점수 범위 제한 | 최소: 0, 최대: 100 |
| 직접 수식 (고급) | 변수 기반 수식 직접 입력 | score * 0.6 + bonus |

- 각 템플릿은 한글 가이드 + 예시가 포함된 입력 폼 제공
- 직접 수식은 사용 가능한 변수 목록(score, maxScore, count 등)을 드롭다운으로 제공
- 저장 전 테스트 값으로 결과 검증 필수
- **보안**: 직접 수식은 허용 변수/함수 화이트리스트만 사용 가능, 서버에서 샌드박스 환경으로 실행 (코드 인젝션 방지)

### A. 위원 점수 집계형
- **대상**: 서류, 면접, 실기
- **입력**: 평가위원의 항목별 점수 또는 등급 (엑셀 업로드)
- **특징**:
  - 면접일자/조별로 위원 수가 다를 수 있음 (최대 위원 수 기준 열 생성, 적은 날은 빈칸)
  - 항목 구조: 대항목 → 소항목(optional) 계층 지원
  - 계산 과정을 파이프라인 빌더로 자유롭게 구성

### B. 자동 채점형 (필답고사)
- **대상**: 필답고사
- **입력**: 정답지 + 수험생 마킹 답안
- **처리**: 정답 대조 → 문항별 배점 적용 → 총점 → 환산
- **특이사항**:
  - 복수정답 지원
  - 출제 오류 처리 (전원 정답 처리 / 배점 제외 선택)
  - 시험유형별(A형/B형 등) 정답지·배점 분리
  - 채점 결과 검증용 감사 로그
  - 과목이 여러 개인 경우: 과목별 업로드 or 통합 업로드 선택

### C. 문항 점수 계산형 (논술)
- **대상**: 논술
- **입력**: 문항별 점수 (사람이 채점한 결과)
- **처리**: 계산 기준 적용 → 환산

### D. 매핑 테이블형
- **대상**: 자격증, 공인어학, 전적대학성적, 실적, (비교과 — 출결/봉사 등 필요 시)
- **입력**: 유형에 따라 다름
  - 자격증: 자격유형 + 급수
  - 공인어학: 어학유형 + 점수 또는 등급
  - 전적대학: 백분위 또는 평균평점
  - 실적: 실적유형 + 등급/수준
- **처리**: 대학별로 설정한 매핑 테이블에서 점수 조회
- **설정**: 매핑 테이블 편집기 (행 추가/삭제로 조건 → 점수 매핑 관리)

## 화면 구성

### 1. 회원가입 / 로그인
- 회원가입: 대학 도메인 이메일 또는 초대 코드로 가입
- 이메일 인증 후 해당 대학 공간에 자동 소속
- 로그인: 이메일 + 비밀번호

### 2. 플랫폼 관리자 - 대학 공간 관리
- 대학 공간 생성 (대학명, 허용 이메일 도메인, 초대 코드)
- 소속 사용자 관리

### 3. 대시보드
- 내 대학의 평가 목록
- 필터: 학년도, 전형명 (평가 생성 시 선택 필드로 입력, 선택사항)

### 4. 평가 생성
- 평가 유형 선택 (A/B/C/D)
- 평가명 입력 (예: "2026학년도 수시 면접")
- 학년도 선택 (optional — 그룹핑/필터용)
- 전형명 입력 (optional — 그룹핑/필터용)
- **기존 평가 복사** 기능: 이전 평가를 선택하면 설정을 그대로 복사하여 수정 가능
- 유형 선택 후 해당 유형의 설정 화면으로 진입

### 5. 평가 세부 설정 (유형별)

#### A. 위원 점수 집계형 설정

##### 기본 정보
- 평가명
- 최대 평가위원 수 (엑셀 열 수 결정, 실제 위원 수는 행마다 다를 수 있음)

##### 위원 데이터 유형 (엑셀에 들어오는 값의 형태)
- **점수**: 위원이 항목별 점수를 매긴 데이터
- **등급**: 위원이 항목별 등급(A/B/C 등)을 매긴 데이터
  - 항목별 등급→점수 매핑 테이블 설정 (항목마다 다를 수 있음)
  - 예: 논리력 A=50, B=40, C=30 / 인성 A=100, B=80, C=60

##### 항목 구조
- **대항목** 추가/삭제 (예: 인성, 전공적합성)
- 대항목별 **소항목** 추가/삭제 — optional (예: 인성 → 성실성, 협동심)
- 소항목 없이 대항목만 사용 가능
- 항목별 설정: 만점, 가중치, 과락 기준

##### 계산 파이프라인 빌더 (핵심)
계산 과정을 단계별 연산 블록 조합으로 구성한다.
관리자가 [단계 추가] 버튼으로 연산을 순서대로 쌓는 방식.

**데이터 형태 변화:**
초기 데이터는 [항목 × 위원] 2차원 행렬. 파이프라인은 이 행렬을 단계적으로 축소하여 최종 단일 점수를 산출한다.

```
초기: [항목 × 위원] 행렬
         논리력  표현력  인성
위원1:    40     30     50
위원2:    45     35     45
위원3:    35     25     55
```

**경로1 — 위원 총점 기준 (위원 차원 먼저 축소)**
위원별로 항목을 합산한 뒤, 위원 총점을 비교/집계
```
[위원별 항목 합산] → [120, 125, 115]  (위원별 총점)
→ [최고 위원 제외] → [120, 115]       (125점 위원 제거)
→ [위원 평균] → 117.5                 (단일 점수)
```

**경로2 — 항목별 기준 (항목 차원 유지하며 위원 축소)**
각 항목별로 위원 점수를 집계한 뒤, 항목을 합산
```
[항목별 최고점 제외] → 논리력[40,35], 표현력[30,25], 인성[50,45]
→ [항목별 위원 평균] → 논리력37.5, 표현력27.5, 인성47.5
→ [항목 합산] → 112.5                 (단일 점수)
```

**사용 가능한 연산 블록:**

*전처리 블록*

| 블록 | 입력 | 출력 | 설명 |
|------|------|------|------|
| 등급→점수 변환 | [항목×위원] 등급 | [항목×위원] 점수 | 항목별 매핑 테이블로 점수 변환 |

*경로1 블록 — 위원 총점 기준*

| 블록 | 입력 | 출력 | 설명 |
|------|------|------|------|
| 위원별 항목 합산 | [항목×위원] | [위원 총점 배열] | 각 위원의 모든 항목 점수를 합산 |
| 위원별 항목 가중합산 | [항목×위원] | [위원 총점 배열] | 각 위원의 항목 점수에 가중치 적용 후 합산 |
| 가상 위원 추가 | [위원 총점 배열] | [위원 총점 배열 +1] | 가상 점수 생성 추가 (최고최저 평균 / 전체 평균 / 지정값) |
| 최고 위원 제외 | [위원 총점 배열] | [위원 총점 배열 -1] | 총점이 가장 높은 위원 제거 |
| 최저 위원 제외 | [위원 총점 배열] | [위원 총점 배열 -1] | 총점이 가장 낮은 위원 제거 |
| 위원 평균 | [위원 총점 배열] | 단일 점수 | 남은 위원 총점의 평균 |
| 위원 합산 | [위원 총점 배열] | 단일 점수 | 남은 위원 총점의 합산 |

*경로2 블록 — 항목별 기준*

| 블록 | 입력 | 출력 | 설명 |
|------|------|------|------|
| 항목별 가상 위원 추가 | [항목×위원] | [항목×위원 +1] | 각 항목에 대해 가상 위원 점수 추가 |
| 항목별 최고점 제외 | [항목×위원] | [항목×위원 -1] | 각 항목에서 최고 위원 점수 제거 |
| 항목별 최저점 제외 | [항목×위원] | [항목×위원 -1] | 각 항목에서 최저 위원 점수 제거 |
| 항목별 위원 평균 | [항목×위원] | [항목별 점수] | 각 항목의 위원 점수를 평균 |
| 항목별 위원 합산 | [항목×위원] | [항목별 점수] | 각 항목의 위원 점수를 합산 |
| 가중치 적용 | [항목별 점수] | [항목별 점수] | 항목별 가중치 반영 |
| 항목 합산 | [항목별 점수] | 단일 점수 | 항목 점수 합산 |
| 항목 평균 | [항목별 점수] | 단일 점수 | 항목 점수 평균 |

*소항목 블록 (소항목이 있는 경우)*

| 블록 | 입력 | 출력 | 설명 |
|------|------|------|------|
| 소항목 합산→대항목 | [소항목별 점수] | [대항목별 점수] | 소항목 합산으로 대항목 산출 |
| 소항목 가중합산→대항목 | [소항목별 점수] | [대항목별 점수] | 소항목 가중합산으로 대항목 산출 |

*후처리 블록*

| 블록 | 입력 | 출력 | 설명 |
|------|------|------|------|
| 항목별 과락 판정 | [항목×위원] 또는 [항목별 점수] | 동일 + 과락 플래그 | 항목별 과락 기준 체크 (경로1 진입 전 또는 원본 데이터 기반으로 판정) |
| 전체 과락 판정 | 단일 점수 | 동일 + 과락 플래그 | 전체 점수 기준 과락 체크 |
| 만점 기준 환산 | 단일 점수 | 단일 점수 (원점수) | 100점 만점 기준으로 환산 → 원점수 산출 |
| 환산 만점 적용 | 단일 점수 (원점수) | 단일 점수 (환산점수) | 환산 만점 입력 (예: 500) → 원점수 × (환산만점 / 100) = 환산점수 (예: 85 × 5 = 425점) |

**위원 수별 조건부 파이프라인:**
면접일자별로 위원 수가 다를 수 있으므로, 위원 수별로 다른 파이프라인을 설정 가능.
시스템이 각 행의 실제 위원 수(빈칸이 아닌 셀 수)를 자동 감지하여 해당 파이프라인 적용.

```
위원 5명일 때 (경로1):
  [위원별 항목 합산] → [최고 위원 제외] → [최저 위원 제외] → [위원 평균] → [전체 과락 판정] → [만점 기준 환산] → [환산 만점 적용]

위원 4명일 때 (경로1):
  [위원별 항목 합산] → [가상 위원 추가(최고최저 평균)] → [최고 위원 제외] → [최저 위원 제외] → [위원 평균] → [전체 과락 판정] → [만점 기준 환산] → [환산 만점 적용]

위원 3명일 때 (경로2):
  [항목별 위원 평균] → [가중치 적용] → [항목 합산] → [전체 과락 판정] → [만점 기준 환산] → [환산 만점 적용]
```

4명 구체 예시 (경로1):
```
         논리력  표현력
위원1:    40     30     → 합계 70
위원2:    45     35     → 합계 80
위원3:    35     25     → 합계 60
위원4:    50     20     → 합계 70

[위원별 항목 합산] → [70, 80, 60, 70]
[가상 위원 추가(최고최저 평균)] → (80+60)/2 = 70 → [70, 80, 60, 70, 70]
[최고 위원 제외] → 80 제거 → [70, 60, 70, 70]
[최저 위원 제외] → 60 제거 → [70, 70, 70]
[위원 평균] → 70
```

**블록 유효성 검증:**
각 블록은 입력/출력 데이터 형태가 정의되어 있으므로, 이전 블록의 출력과 다음 블록의 입력이 맞지 않으면 경고/차단.
- 경로1 블록과 경로2 블록은 혼용 불가 (입력 데이터 형태가 다름)
- [만점 기준 환산] 블록이 파이프라인에 없으면 경고
- 등급 데이터인데 [등급→점수 변환] 없이 진행하면 경고
- 최종 출력이 단일 점수가 아니면 경고

##### 샘플 미리보기
- 파이프라인 설정 후 샘플 점수를 입력하면 각 단계별 중간 결과와 최종 결과를 확인

#### B. 자동 채점형 설정
- 기본 정보: 평가명, 과목 수, 만점
- 과목별 설정:
  - 과목명, 문항 수, 문항별 배점, 과목 만점
  - 과목별 과락 기준
  - **시험유형** 설정 (예: A형, B형) — 유형별로 정답지와 배점이 다를 수 있음
    - 유형 간 문항 수가 다를 경우: 유형별 별도 시트로 업로드
  - 유형별 정답지 등록 (문항별 정답, 복수정답 여부)
  - **출제 오류 처리**: 특정 문항에 대해 사후 처리 가능
    - 전원 정답 처리: 해당 문항을 모든 수험생 정답으로 간주
    - 배점 제외: 해당 문항을 총점 계산에서 제외하고 나머지 기준으로 재산출
- 전체 과락 기준
- **계산 파이프라인 빌더**: 채점 → 과목 합산 → 과락 판정 → 만점 기준 환산 → 환산 만점 적용 등 블록 조합
  - ※ 블록별 입력/출력 상세 정의는 기술 설계 단계에서 A유형과 동일한 형식으로 정의
- 샘플 미리보기

#### C. 문항 점수 계산형 설정
- 기본 정보: 평가명, 만점
- **채점위원 수** (optional): 복수 채점위원이 같은 문항을 채점하는 경우
  - 위원별 점수를 받아 집계 (평균 / 최고최저 제외 등 — A유형과 동일한 위원 집계 블록 재활용)
  - 채점위원 없이 단일 점수만 받는 것도 가능
- 문항 구조: **대문항 → 소문항** 계층 지원
  - 대문항 추가/삭제 (예: 1번, 2번, 3번)
  - 대문항별 소문항 추가/삭제 (예: 1-1, 1-2, 1-3)
  - 소문항 없이 대문항만 사용도 가능
- 문항별 설정: 만점, 배점, 가중치
- 과락 설정:
  - 소문항별 과락 기준 (optional)
  - 대문항별 과락 기준 (optional)
  - 전체 과락 기준
- 대문항 점수 산출: 소문항 합산 / 가중합산
- 전체 계산 기준 (합산 / 가중합산 — 특수 계산은 커스텀 블록으로 대체)
- **계산 파이프라인 빌더**: 위원 집계(해당 시) → 소문항 합산 → 대문항 합산 → 과락 판정 → 만점 기준 환산 → 환산 만점 적용 등 블록 조합
  - ※ 블록별 입력/출력 상세 정의는 기술 설계 단계에서 A유형과 동일한 형식으로 정의
- 샘플 미리보기

#### D. 매핑 테이블형 설정
- 기본 정보: 평가명, 매핑 유형 선택 (자격증/어학/전적대학/실적/기타)
- 매핑 테이블 편집:
  - 자격증: (유형, 급수) → 점수 테이블
  - 공인어학: (어학유형, 점수구간) → 점수 테이블
  - 전적대학: (평점/백분위 구간) → 점수 테이블
  - 실적: (유형, 등급) → 점수 테이블
  - 기타: 범용 (조건1, 조건2) → 점수 테이블
- 매핑 테이블 입력 방식:
  - **엑셀 업로드**: 매핑 테이블을 엑셀로 업로드 (행이 많을 때 편리)
  - **화면 직접 편집**: 업로드 후 화면에서 확인/수정 가능
  - 두 방식 병행 — 업로드로 초기 데이터 구성, 화면에서 미세 조정
- 만점 설정
- 전체 과락 기준
- **매칭 실패 처리**: 매핑 테이블에 없는 값이 업로드되면 해당 행을 오류로 표시, 업로드 시 실패 인자 알람
- **복수 보유 시**: 사용자가 인정한 건만 업로드 (엑셀에 1건만 기재, 복수 건 선별은 업로드 전 대학 측에서 처리)
- **계산 파이프라인 빌더**: 매핑 조회 → 과락 판정 → 만점 기준 환산 → 환산 만점 적용 등 블록 조합
  - ※ 블록별 입력/출력 상세 정의는 기술 설계 단계에서 A유형과 동일한 형식으로 정의
- 샘플 미리보기

### 6. 엑셀 양식 및 업로드

#### 양식 자동 생성 (유형별)

**A. 위원 점수 집계형** (예: 최대 위원 5명, 대항목 2개, 소항목 있음)
```
수험번호 | 성명 | 인성_성실성_위원1 | 인성_성실성_위원2 | ... | 인성_성실성_위원5 | 인성_협동심_위원1 | ... | 전공_소항목1_위원1 | ...
```
- 위원 수가 적은 행은 뒷 열을 빈칸으로 남김 → 시스템이 빈칸 수로 실제 위원 수 자동 감지 → 해당 위원 수 파이프라인 적용
- 면접조 열 불필요 (위원 수 기반 자동 분기)
- 등급 입력 시 셀 값은 A, B, C 등

**B. 자동 채점형 — 유형 동일 문항 수** (예: 과목 1개, 문항 50개)
```
수험번호 | 성명 | 시험유형 | 1번 | 2번 | 3번 | ... | 50번
```

**B. 자동 채점형 — 유형별 문항 수 다름** (유형별 별도 시트)
```
[A형 시트] 수험번호 | 성명 | 1번 | 2번 | ... | 50번
[B형 시트] 수험번호 | 성명 | 1번 | 2번 | ... | 45번
```

**C. 문항 점수 계산형 — 단일 채점** (예: 대문항 2개, 소문항 포함)
```
수험번호 | 성명 | 1-1 | 1-2 | 1-3 | 2-1 | 2-2
```

**C. 문항 점수 계산형 — 복수 채점위원** (예: 위원 2명, 다중행)
```
수험번호 | 성명 | 위원 | 1-1 | 1-2 | 1-3 | 2-1 | 2-2
001     | 홍길동 | 위원1 | 13  | 14  | 8   | 25  | 28
001     | 홍길동 | 위원2 | 12  | 15  | 9   | 27  | 26
002     | 김철수 | 위원1 | 10  | 12  | 7   | 20  | 22
002     | 김철수 | 위원2 | 11  | 13  | 8   | 22  | 24
```

**D. 매핑 테이블형** (예: 자격증)
```
수험번호 | 성명 | 자격유형 | 급수
```

#### 업로드 프로세스
- 양식 다운로드 → 데이터 입력 → 업로드
- 업로드 시 유효성 검증 (필수 컬럼, 데이터 형식, 점수 범위)
- 검증 결과 미리보기 (오류 행 하이라이트) → 확인 후 저장

#### 재업로드 정책
- 같은 평가에 다시 업로드 시 **기존 데이터를 덮어쓰기**
- 이전 업로드 이력은 내부적으로 보관 (업로드 일시, 파일명, 데이터)
- 필요 시 이전 버전으로 **롤백 가능**

### 7. 계산 및 결과 조회

#### 계산 실행
- **수동 계산 방식**: "계산" 버튼을 눌러야 결과 산출 (자동 계산 아님)
- 계산 완료 후 설정 또는 데이터에 변경사항이 생기면:
  - 경고 배지 표시 ("설정이 변경되었습니다. 재계산이 필요합니다")
  - 재계산 전까지는 이전 계산 결과를 유지하되, 변경됨을 명시

#### 결과 정의
- **원점수**: 모든 집계를 마친 100점 만점 기준 점수
- **환산점수**: 원점수에 환산 만점을 적용한 점수 (예: 원점수 85 × 5 = 환산점수 425, 면접 500점 만점)

#### 결과 조회
- 지원자별 원점수 + 환산점수 + 과락 여부 테이블
- **중간 결과 확인**: 파이프라인 각 단계별 중간 점수를 펼쳐 볼 수 있음 (검증용)
  - 예: 위원별 점수 → 위원 집계 후 → 항목 합산 후 → 최종 환산
- 필터링, 정렬 (면접조, 과락 여부 등)
- **엑셀 다운로드** (중간 결과 포함 옵션)
- **출력물** (추후 정의)

     3명: [위원별 항목 합산] → [위원 평균] → [전체 과락 판정] → [만점 기준 환산] → [환산 만점 적용]

## 공통 기능
- **평가 복사**: 기존 평가의 모든 설정을 복사하여 새 평가 생성 (매년 유사한 평가 재사용)
- **샘플 미리보기**: 모든 유형에서 설정 완료 후 샘플 데이터로 각 단계별 중간 결과 + 최종 결과 검증
- **업로드 이력 관리**: 업로드 이력 보관, 이전 버전 롤백 가능
- **소수점 처리 설정**: 파이프라인 블록 단위로 세분화 설정 가능
  - 처리 방식: 반올림 / 버림(절사) / 올림
  - 소수점 자릿수: 0자리(정수) / 1자리 / 2자리 / 3자리
  - 적용 단위:
    - **기본값**: 평가 전체에 하나의 소수점 규칙 설정 (단순)
    - **블록별**: 특정 블록에 개별 소수점 규칙 지정 가능 (예: 위원 평균은 소수점 2자리 반올림, 최종 환산은 정수 버림)
  - 블록별 설정이 없으면 기본값 적용

## 검증
- A유형: 위원 수별 조건부 파이프라인 동작 검증, 등급 입력 + 소항목 계층 검증
- B유형: 복수정답, 출제오류(전원정답/배점제외), 시험유형별 채점 검증
- C유형: 대문항/소문항 계층 계산 검증
- D유형: 각 매핑 유형별(자격증/어학/전적대학) 변환 검증
- 공통: 소수점 처리, 재업로드 롤백, 평가 복사, 원점수+환산점수 산출 검증
