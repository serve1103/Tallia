# Tallia — PRD (Product Requirements Document)

## 1. 프로젝트 개요

### 제품명
Tallia (탈리아)

### 한 줄 요약
대학 입학처에서 지원자 평가점수를 원점수(100점) 및 환산점수로 산출하는 멀티테넌트 웹 애플리케이션

### 배경 및 목적
- 대학별로 평가 항목, 환산 공식, 점수 처리 방식이 모두 다름
- 기존 솔루션은 원점수를 환산하는 기능이 없거나 부족함
- Tallia는 다양한 평가 유형(면접, 필답, 논술, 자격증 등)에 대해 유연한 점수 환산 파이프라인을 제공하여 이 문제를 해결

### 핵심 가치
- **유연성**: 파이프라인 빌더로 대학별 다양한 계산 방식을 설정으로 해결
- **정확성**: 블록 단위 소수점 처리, 중간 결과 검증, 감사 로그
- **편의성**: 엑셀 기반 데이터 입출력, 양식 자동 생성, 평가 복사

## 2. 범위

### In Scope (MVP — Phase 1)
- 멀티테넌트 대학 공간 관리
- 회원가입/로그인 (도메인 이메일 + 초대 코드)
- 4가지 평가 유형 전부 포함:
  - A. 위원 점수 집계형 (서류, 면접, 실기)
  - B. 자동 채점형 (필답고사)
  - C. 문항 점수 계산형 (논술)
  - D. 매핑 테이블형 (자격증, 어학, 전적대학)
- 계산 파이프라인 빌더 (모든 유형)
- 커스텀 블록 (템플릿 + 고급 수식)
- 엑셀 양식 자동 생성 / 업로드 / 검증
- 원점수 + 환산점수 결과 조회 / 중간 결과 확인
- 결과 엑셀 다운로드
- 재업로드 / 롤백
- 평가 복사
- 소수점 처리 (기본값 + 블록별)
- 환산 만점 설정
- 감사 로그 (데이터 접근/수정/다운로드 이력 + B유형 채점 검증 로그)

### Out of Scope
- 최종 점수 산출 (전형별 합산 — 다른 프로젝트에서 처리)
- 학생부 교과 / 비교과 / 수능 성적 환산 (추후 플러그인 확장)
- 출력물 (추후 정의)
- 평가위원이 직접 웹에서 점수 입력하는 기능

### 추후 확장 (Phase 2+)
- 학생부 교과/비교과 환산 유형
- 수능 성적 환산 유형
- 출력물 (인쇄용 보고서)
- API 연동 (다른 프로젝트와 자동 데이터 전달)

## 3. 전체 사용 흐름

```
1. 플랫폼 관리자가 대학 공간 생성 (허용 이메일 도메인 + 초대 코드 설정)
2. 대학 사용자가 도메인 이메일 또는 초대 코드로 회원가입
3. 평가 생성 시 유형 선택 (A/B/C/D)
4. 유형별 세부 설정 + 계산 파이프라인 구성
5. 설정 기반 엑셀 양식 자동 생성 → 다운로드
6. 관리자가 양식에 데이터 채워서 업로드
7. "계산" 버튼으로 수동 실행
8. 원점수 + 환산점수 결과 조회 / 엑셀 다운로드
```

## 4. 사용자 역할

| 역할 | 설명 | 주요 활동 |
|------|------|-----------|
| 플랫폼 관리자 | 시스템 운영자 | 대학 공간 생성/관리, 도메인/초대코드 설정 |
| 대학 관리자 | 입학처 직원 | 평가 설정, 파이프라인 구성, 엑셀 업로드, 결과 조회/다운로드 |

## 5. 화면 구성 요약

| 화면 | 설명 |
|------|------|
| 회원가입/로그인 | 도메인 이메일 또는 초대 코드로 가입, 이메일 인증 |
| 대학 공간 관리 | 플랫폼 관리자용 — 대학 생성, 도메인/초대코드, 사용자 관리 |
| 대시보드 | 평가 목록 + 학년도/전형명 필터 |
| 평가 생성 | 유형 선택(A/B/C/D), 평가명, 학년도/전형명(optional), 기존 평가 복사 |
| 평가 세부 설정 | 유형별 설정 + 파이프라인 빌더 + 공용 설정(환산 만점, 소수점 처리) |
| 엑셀 업로드 | 양식 다운로드, 파일 업로드, 검증 미리보기, 업로드 이력/롤백 |
| 결과 조회 | 계산 실행, 원점수+환산점수 테이블, 중간 결과 펼침, 엑셀 다운로드 |

> 상세 UI는 [wireframe.html](./wireframe.html) 참조

## 6. 결과 정의

- **원점수**: 모든 집계를 마친 100점 만점 기준 점수
- **환산점수**: 원점수에 환산 만점을 적용한 점수 (예: 원점수 85 × 5 = 환산점수 425, 면접 500점 만점)

## 7. 주요 정책

### 계산 실행
- **수동 계산 방식**: "계산" 버튼을 눌러야 결과 산출 (자동 계산 아님)
- 계산 완료 후 설정/데이터 변경 시 경고 배지 표시, 재계산 안내

### 재업로드
- 같은 평가에 다시 업로드 시 기존 데이터 덮어쓰기
- 이전 업로드 이력 보관 (업로드 일시, 파일명, 데이터)
- 필요 시 이전 버전으로 롤백 가능

### 소수점 처리
- 처리 방식: 반올림 / 버림(절사) / 올림
- 소수점 자릿수: 0~3자리
- 적용 단위: 평가 전체 기본값 + 블록별 개별 설정 가능

## 8. 비기능 요구사항

### 성능
- 동시 접속: 50개+ 대학, 대학당 평균 3~5명 → 최대 250명 동시 접속
- 엑셀 업로드: 최대 10,000행 / 10MB 파일 지원
- 계산 처리: 1,000명 데이터 기준 10초 이내
- 엑셀 다운로드: 10,000행 기준 30초 이내
- ※ 성능 수치는 목표값이며, 기술 설계 및 부하 테스트 후 재검토

### 보안
- 비밀번호: bcrypt 해싱
- 인증: JWT 기반
- 멀티테넌트 격리: tenant_id 기반, 다른 대학 데이터 접근 불가
- 커스텀 블록 수식: 허용 변수/함수 화이트리스트, 서버 샌드박스 실행
- HTTPS 필수

### 개인정보 보호
- 수험생 데이터(수험번호, 성명, 점수)는 개인정보 보호법 적용 대상
- 데이터 보관 기간: 대학별로 설정 가능 (기본 3년, 이후 자동 삭제 또는 관리자 수동 삭제)
- 접근 로그: 모든 데이터 조회/수정/다운로드에 대해 감사 로그 기록
- 엑셀 다운로드 시 다운로드 이력 기록
- 데이터 삭제 시 복구 불가능하도록 완전 삭제

### 가용성
- 입학 시즌(11~2월) 안정적 운영 필수
- 일반 SLA: 99.5% (월간)
- 정기 점검: 입학 시즌 외 시간대

### 지원 환경
- **데스크톱**: Chrome, Edge (최신 2개 버전)
- **모바일**: 반응형 웹 (결과 조회 중심)
- IE 미지원

## 9. 설계 원칙

- **파이프라인 플러그인 구조**: 평가 유형(A/B/C/D)을 독립 모듈로 설계, 추후 새 유형 추가 시 기존 코드 변경 최소화
- **계산 파이프라인 빌더**: 모든 유형의 계산 과정을 연산 블록 조합으로 구성, A유형은 위원 수별 조건부 파이프라인 지원
- **D(매핑 테이블)의 균형**: 범용 테이블 편집기 + 자주 쓰는 유형은 프리셋 UI 제공
- **B(자동 채점)의 신뢰성**: 채점 결과 감사 로그, 복수정답/출제오류 처리 필수

## 10. MVP 구현 우선순위

전 유형(A/B/C/D)을 MVP에 포함하되, 구현 순서는 의존성과 복잡도 기준:

| 순서 | 대상 | 이유 |
|------|------|------|
| 1 | 인프라 + 인증 + 대학 공간 | 모든 유형의 기반 |
| 2 | 파이프라인 빌더 엔진 (공통) | 모든 유형이 사용하는 핵심 모듈 |
| 3 | A. 위원 점수 집계형 | 가장 복잡, 파이프라인 빌더의 전체 기능 검증 |
| 4 | D. 매핑 테이블형 | 가장 단순, 빠르게 완성 가능 |
| 5 | B. 자동 채점형 | 독립적 기능 (정답 대조 엔진) |
| 6 | C. 문항 점수 계산형 | A와 구조 유사, A 완성 후 빠르게 확장 |
| 7 | 공통 기능 (복사, 이력, 소수점) | 유형별 기능 완성 후 마감 |
| 8 | 결과 조회 + 엑셀 다운로드 | 최종 산출물 |

## 11. 외부 연동

- 최종 점수 산출은 다른 프로젝트에서 처리
- Phase 1: Tallia 결과를 **엑셀 다운로드 후 수동 업로드**
- Phase 2+: **API 연동** 확장 가능하도록 설계 (RESTful API)
